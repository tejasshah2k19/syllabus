1. Basic Navigation & File Commands

Where am I? pwd Prints "/home/ec2-user"
Listing Files ls -la Lists all files (hidden too) with details like permissions.
ls -lh Lists files with human-readable sizes (MB, GB).
Directories cd /var/www Change directory to /var/www.
mkdir -p app/src Creates parent and child directories at once.
Manipulation cp -r source_folder dest_folder Copy a folder recursively.
rm -rf my_folder Force remove a folder (Careful!).
mv old.txt new.txt Rename or move a file.

2. File Viewing & Editing

Editors nano config.conf Easy terminal editor. (Ctrl+O to save, Ctrl+X to exit).
vi / vim Advanced editors. (Press 'i' to insert, ':wq' to save/quit).
Viewing Content cat file.txt Dumps whole file content.
head -n 5 file.txt Shows first 5 lines.
tail -f /var/log/messages Follows the end of a file in real-time (Great for logs).
Search & Filter grep -r "error" . Recursively search for "error" in current directory.
grep -i "failed" log.txt Case-insensitive search.

3. User & Permission Management

Super User sudo command Run command as root.
sudo su - Switch to root user shell entirely.
Permissions (chmod) chmod 400 keypair.pem Read-only by owner (Required for SSH keys).
chmod 755 script.sh Owner can write; everyone else can read/execute.
chmod +x script.sh Make a file executable.
Ownership (chown) chown ec2-user:apache index.html Change owner to ec2-user and group to apache.

4. Networking Commands

Connectivity ping google.com Check internet connection.
curl http://localhost Test if your web server is responding locally.
wget https://example.com/file.zip Download a file from the web.
Ports & IP netstat -tulpn Show all listening ports (Is port 80 open?).
ifconfig or ip a Show IP address configuration.
Remote Access ssh -i key.pem user@ip Secure Shell connection.
scp -i key.pem file.txt user@ip:~ Secure Copy file to remote server.

5. System Monitoring & Processes

Resources top or htop Real-time view of CPU/RAM usage.
df -h Check disk space usage (human readable).
free -h Check available RAM.
Processes ps aux | grep java Find a running Java process.
kill -9 1234 Force kill process with PID 1234.

6. Package Management

Command depends on your OS (Amazon Linux/RHEL vs Ubuntu).

Amazon Linux / CentOS (yum) yum update -y Update all packages.
yum install httpd -y Install Apache web server.
yum remove httpd Uninstall a package.
Ubuntu / Debian (apt) apt update && apt upgrade Refresh repo list and upgrade packages.
apt install apache2 Install Apache.

7. Systemctl Service Management

Control background services like Web Servers or Databases.

systemctl start httpd Start the service immediately.
systemctl stop httpd Stop the service.
systemctl restart httpd Restart (useful after config changes).
Important for AWS systemctl enable httpd Ensure service starts automatically on reboot!
systemctl status httpd Check if it's running or failed.

8 & 9. Archives & Logs

Zipping/Tar tar -cvf backup.tar /var/www Create a tar archive of a folder.
tar -xvf backup.tar Extract a tar archive.
zip -r code.zip folder/ Zip a folder recursively.
Log Files (Debugging) tail -f /var/log/messages System logs (Amazon Linux).
tail -f /var/log/syslog System logs (Ubuntu).
tail -f /var/log/nginx/error.log Web server errors.

11 & 12. Git & Docker

Git git clone https://github.com/user/repo.git Download code.
git pull Update local code from remote.
git commit -m "msg" Save changes locally.
Docker docker ps List running containers.
docker build -t myapp . Build an image from Dockerfile.
docker run -p 80:80 myapp Run container on port 80.

13. AWS CLI

Managing AWS resources directly from the Linux terminal.

Setup aws configure Enter Access Key, Secret Key, Region.
S3 (Storage) aws s3 ls List all buckets.
aws s3 cp file.txt s3://mybucket/ Upload file to S3.
aws s3 sync . s3://mybucket/ Sync current folder to S3.
EC2 aws ec2 describe-instances Get JSON details of your instances.
