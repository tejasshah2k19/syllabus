Course Curriculum

Comprehensive 20-Module Program

Linux Fundamentals

01
  • Filesystem Hierarchy: Understanding basic Linux structure.
  • Commands: Basic commands, Grep, Filter commands (head, tail, more, less).
  • Editors: Vim editor modes and usage.
  • User Management: Creating/modifying users, groups, and permissions.
  • Package Management: Yellowdog Updater Modified (YUM) and RPM.

Cloud Computing Concepts

02
  • Why Cloud?: Features, benefits, and cost models.
  • Deployment Models: Public, Private, and Hybrid Cloud.
  • Service Models: IaaS, PaaS, and SaaS explained.
  • Virtualization: Hypervisors and their role in the cloud.
  • Scaling: Importance of scaling and various applications.

AWS Overview

03
  • Global Infrastructure: Regions and Availability Zones.
  • Service Categories: Compute, Storage, Network, and Database services.
  • Getting Started: Free Tier account creation and Management Console tour.
  • Marketplace: Overview of AWS Marketplace features.

IAM (Identity Access Management)

04
  • Authentication: Password policies and MFA.
  • Users & Groups: Creating accounts and associating policies.
  • Roles: Creating roles for service-to-service access.
  • Access: Programmatic vs. Console access management.

EC2 (Elastic Compute Cloud)

05
  • Instances: Launching Basic EC2 instances from AMI.
  • Pricing Models: On-demand, Reserved, Spot, and Dedicated.
  • Security: Security groups, Tags, and Key Pairs (Public/Private).
  • Storage: Attaching EBS volumes and creating Snapshots.
  • Custom Images: Creating and deploying custom AMIs.

Auto-scaling

06
  • Configuration: Launch configurations for scaling groups.
  • Strategies: Dynamic Scaling and Fixed Number policies.
  • Lifecycle: Understanding the auto-scaling lifecycle.

Load Balancing (ELB)

07
  • Types of ELB: Application, Network, and Classic Load Balancers.
  • Migration: Moving from Classic to newer generation balancers.
  • Components: Listeners, Targets, and Health Checks.

EBS (Elastic Block Storage)

08
  • Volume Management: Creating, attaching, and deleting volumes.
  • Data Protection: Creating and deleting snapshots.
  • Usage: Persistent block storage for EC2.

Object Storage (S3 & Glacier)

09
  • S3 Basics: Buckets, Objects, Upload/Download.
  • Advanced S3: Versioning, Lifecycle Policies, and Transfer Acceleration.
  • Archival: Glacier storage for long-term retention.
  • Hybrid: AWS Storage Gateway and Import/Export.

CloudFront (CDN)

10
  • Distributions: Creating distributions and configuring origins.
  • Performance: Caching behaviors and global edge locations.
  • Security: Implementing global restrictions (Geo-blocking).

Route53 (DNS)

11
  • Hosted Zones: Creating zones and understanding domain names.
  • Routing Policies: Simple, Weighted, Latency, and Failover routing.
  • Hosting: Pointing custom domains to AWS resources.

Security Management

12
  • Shared Responsibility Model: AWS vs. Customer responsibilities.
  • Auditing: Importance of CloudTrail.
  • Optimization: Using Trusted Advisor for security and cost checks.

VPC (Virtual Private Cloud)

13
  • Networking: IP addressing, CIDR, and Subnets (Public/Private).
  • Security: Security Groups vs. Network ACLs.
  • Connectivity: VPN configurations and Internet Gateways.

RDS (Relational Database Service)

14
  • Engines: MySQL, Oracle, MS SQL, etc.
  • Management: Configuration, Backups, and Maintenance Windows.
  • Connectivity: Connecting applications to RDS instances.

DynamoDB (NoSQL)

15
  • Basics: Creating tables and adding data manually.
  • Consistency: Strong vs. Eventual consistency models.
  • Performance: Read/Write capacity units and Alarms.

Monitoring (CloudWatch)

16
  • Metrics: Getting statistics for EC2 and other services.
  • Alarms: Setting thresholds and configuring actions.
  • Integration: Combining CloudWatch with Auto-scaling.

Application Services

17
  • Messaging: SNS (Topics/Subscriptions) and SQS (Queues).
  • Email: SES (Simple Email Service).
  • Compute/App: AWS Lambda (Serverless) and Elastic Beanstalk.

Troubleshooting

18
  • Scenarios: Troubleshooting EC2, ELB, and CloudFront issues.
  • Tools: Using CloudWatch logs for diagnosis.

Architecture & Design

19
  • Resilience: High Availability Design and Disaster Recovery (DR).
  • Best Practices: Backup strategies and DR planning.

DevOps Fundamentals

20
  • Concepts: Understanding DevOps culture.
  • Toolsets: Overview of modern DevOps tools in the cloud.
